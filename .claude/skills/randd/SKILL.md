# 1. Brainstorming Phase

This phase is dedicated to brainstorming. Do NOT implement any solution yet.
Let's set a lean, clear context to design the optiman solution or solve the right problem.

- Ask me for a detailed description of the idea to design or problem to solve.
- Ask me for examples of potential solutions and/or expected outcomes.
- Ask me for the required files and folders to verify their assertions and understand the current state of the codebase.
- Let's brainstorm

**Exit criteria:** You have enough clarity to describe the solution without saying "it depends" or "we could do X or Y."

Share a concise but detailed summary or the findings with the user.
Confirm with the user if mutual understanding is reached before going to the design phase.

# 2. Design Phase

- Describe the mission critical components to build/modify:
    - Component responsibilities and boundaries
    - Key interfaces between components
    - Key data models
    - Data flow through the system
- Sketch a system design diagram using ASCII art to visualize components and their dependencies.
- Sketch workflow diagrams to visualize component actions.
